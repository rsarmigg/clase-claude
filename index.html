1.1 — Los Modelos

Haiku vs Sonnet vs Opus

Modelo Velocidad Inteligencia Mejor para
Haiku Ultra rápido Buena Tareas simples, clasificación, chat rápido
Sonnet Rápido Muy alta Uso diario, código, análisis, escritura
Opus Moderado Máxima Razonamiento complejo, investigación profunda

Regla general: empieza con Sonnet para el 80% de las tareas. Escala a Opus solo cuando necesites razonamiento profundo.

1.2 — El Arte del Prompting

Framework CRAFT

C Contexto — ¿Quién eres? ¿Cuál es la situación?
R Rol — ¿Qué papel debe asumir Claude?
A Acción — ¿Qué debe hacer exactamente?
F Formato — ¿Cómo debe estructurar la respuesta?
T Tono — ¿Qué voz o estilo debe usar?

1.2 — CRAFT en Acción

Antes y después

Antes
"Escríbeme algo sobre bases de datos"
Después (CRAFT)
"[C] Soy CTO de una startup con 10M eventos/día.
[R] Actúa como arquitecto senior de datos.
[A] Compara PostgreSQL vs MongoDB para analytics en tiempo real.
[F] Tabla comparativa + párrafo de recomendación.
[T] Directo y técnico."

La diferencia no es solo mejor output — es output útil y accionable.

1.3 — Dinámica de Conversación

Claude recuerda todo el hilo

  • Multi-turn context: cada mensaje ve toda la conversación anterior
  • Correcciones: "No, me refiero a..." / "Ajusta el tono a..."
  • Refinamiento iterativo: pide cambios incrementales, no reescribas todo
  • Context window: ~200K tokens (~150K palabras). Si la conversación es muy larga, abre una nueva

Pro tip: Usa "Haz lo mismo pero para..." para reutilizar el formato de una respuesta anterior con diferente contenido.

1.3 — Extended Thinking

Cuando necesitas que Claude piense de verdad

Sin Extended Thinking

Claude responde inmediatamente con su "primera intuición". Ideal para tareas directas y rápidas.

Con Extended Thinking

Claude razona paso a paso internamente antes de responder. Visible como un bloque desplegable. Mejor para problemas complejos, matemáticas, lógica y análisis.

Se activa en el selector de modelo o Claude lo usa automáticamente cuando lo necesita.

3.2 — Anatomía de un Project

Los tres componentes

1. Project Instructions (System Prompt)

Instrucciones que Claude sigue en cada conversación del proyecto. Define tono, formato, reglas, restricciones.

2. Knowledge Base

Sube PDFs, código, docs, guías de estilo. Claude los consulta automáticamente cuando son relevantes.

3. Conversaciones

Cada chat dentro del proyecto hereda las instrucciones y tiene acceso a la knowledge base.

3.3 — Patrones para Equipos

Ideas de Projects que transforman el trabajo

Brand Voice

Sube tu guía de marca. Cada texto generado sigue el tono exacto de tu empresa.

Code Review

Sube tu coding standard. Claude revisa PRs siguiendo tus reglas exactas.

Onboarding

Sube el wiki interno. Nuevos empleados preguntan a Claude en vez de interrumpir al equipo.

En planes Team y Enterprise, los projects se comparten entre miembros del equipo.

  3. Sube al menos un documento (un PDF, un .txt, cualquier referencia)

5.1 — ¿Qué es Claude Code?

Claude en tu terminal

  • CLI tool oficial de Anthropic
  • Lee, entiende y modifica tu codebase
  • Ejecuta comandos en tu terminal
  • Muestra diffs antes de aplicar cambios
  • Entiende dependencias entre archivos
# Instalación

# Uso: navega a tu proyecto y ejecuta
cd mi-proyecto/
claude

Una vez dentro, simplemente escribe lo que necesitas en lenguaje natural.

5.2 — Claude Code en Práctica

Ejemplos reales

Explorar

> "Explica la arquitectura de este proyecto"

Claude lee los archivos clave y genera un resumen de la estructura, dependencias y flujos principales.

Modificar

> "Agrega validación de input al endpoint de registro"

Claude encuentra el archivo correcto, propone los cambios como diff, y espera tu aprobación antes de aplicar.

Crear

> "Crea tests unitarios para el módulo de autenticación"

Claude analiza el código existente, genera tests con los frameworks del proyecto y los ejecuta.

Claude siempre te muestra el diff

5.3 — MCPs Comunes

¿Con qué se puede conectar Claude?

GitHub

Crear issues, PRs, branches. Buscar código. Revisar cambios. Mergear PRs.

Puppeteer

Controlar un browser. Navegar, hacer click, llenar formularios, tomar screenshots.

Bases de Datos

Conectar a PostgreSQL, SQLite, MySQL. Hacer queries, analizar datos.

Slack

Leer mensajes, buscar en canales, enviar notificaciones.

File System

Operaciones avanzadas de archivos más allá de lo básico de Claude Code.

Y muchos más...

Jira, Notion, AWS, GCP, Sentry, Linear, Google Drive, etc.

5.3 — Configuración

Activar un MCP Server

// .claude/settings.json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": [
        "-y", "@anthropic-ai/mcp-server-github"
      ],
      "env": {
        "GITHUB_TOKEN": "ghp_xxxx..."
      }
    },
    "puppeteer": {
      "command": "npx",
      "args": [
        "-y", "@anthropic-ai/mcp-server-puppeteer"
      ]
    }
  }
}

Un archivo JSON, unos pocos líneas, y Claude tiene nuevos superpoderes.

5.4 — Bonus: Hooks

Automatización con hooks

Hooks ejecutan comandos automáticamente en respuesta a eventos de Claude Code 

// settings.json
{
  "hooks": {
    "afterWrite": [
      {
        "command": "npx prettier --write $FILE_PATH",
        "description": "Auto-format después de cada escritura"
      }
    ]
  }
}
  • afterWrite: auto-format, lint, etc.
  • afterCommit: run tests, notify Slack
  • beforeCommand: validaciones de seguridad
